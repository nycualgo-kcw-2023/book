Subtask 3

In this subtask, we can visit every node in the beginning, so we only need to determine the precedence of each node. There are only 18 kinds of nodes, consisting of {+1,+2,…,+9} and {x1,x2,…,x9}.

Suppose there are several nodes {p1​,p2​,⋯,ph​,m1​,m2​,⋯,mk​}, where pi​ denotes the value of node with '+' sign and mj​ denotes the value of node with 'x' sign. There are three possible scenarios to compare two arbitrary nodes:

  • pi​ vs pj​
  • wi​ vs wj​
  • pi​ vs wj​

(Define S as the initial value before doing the below operation)

Scenario 1:

  • If we do pi​ first then pj​, the value will be S+pi​+pj​.
  • If we do pj​ first then pi​, the value will be S+pj​+pi​.

We can find that the two approaches has no difference, indicating that the priority of arbitrary two addition nodes are the same.

Scenario 2:

  • If we do wi​ first then wj​, the value will be S×wi​×wj​.
  • If we do wj​ first then wi​, the value will be S×wj​×wi​.

We can find that the two approaches has no difference, indicating that the priority of arbitrary two multiplication nodes are the same.

Scenario 3:

  • If we do pi​ first then wj​, the value will be (S+pi​)×wj​=S×wj​+pi​×wj​.
  • If we do wj​ first then pi​, the value will be S×wj​+pi​.

We can find that the first approach will always be larger than or equal to the second one, so we can conclude that we always do addition first then do multiplication.

In summary, we can sort all nodes by performing addition followed by multiplication, and thereby identify the maximum value for the solution.

  • Note that the from this subtask on, the answer may exceed 1018, so you may use either double or __int128 to record the current answer, whenever the node value exceed 1018, record the value as 1018 and set the inf flag as true**.

Same as what we do in subtask 3, we can perform some computation:

Subtask 5

In this subtask, each route might have several multiplication nodes, so it might be hard to decide which route to traverse if you only take the first merged multiplication node into consideration (since there might be a better merged multiplication hidden behind). We will introduce a little trick here. Just like what you did to merge addition node into multiplication node, we can also merge multiplication node into another one. Therefore, we can merge all the nodes into a single node in the order of a−1b​. So here’s the question, how to merge two merged multiplication nodes?

  • Merge {×c+d} into {×a+b}

    Which means you do {×a+b} first then do {×c+d}, the value will be

    (si​×a+b)×c+d=si​×(a×c)+(b×c+d)

    That’s right! you will find that you get a node with {×(ac)+(bc+d)}.

After you merge the two nodes, don’t forget to add this new node value into the priority queue to update its order (the comparing value of this node is ac−1bc+d​)

Let’s take the following example:

  • In the beginning, we calculate the comparing value of each node.

  • Merge node of value inf into its parent and reassign the parent of its child to its parent.

  • Recalculate the comparing value of each value and determine which node to be chosen. The merged node will be {×(2×3)+(0×3+9)}={×6+9}.

  • The next node selected should be node 1; however, it has nothing to merge, so we just simply ignore it. Instead, we take {×3+5},{×6+9},{×2} in order. Finally we will get a node

    (((si​×2+4)×3+5)×6+9)×2=si​×72+222

    , which is {×72+111}. Then we can calculate the answer 1×72+222=294.

Subtask 6

In this subtask, each mode might have several children nodes, so we need a data structure to record the children of each node (since if you iterate through all the nodes to get the children every time, you will get TLE). Here, we provide you a solution with adjacency matrix (which is an array of set). When you scan node 1 is the parent of node 2, you will insert 2 into array[1]’s set. This is helpful when you need to update a node’s children to its parent, as you only need to insert the set into its parent’s set.
