F. Bad Sequence

  • Online Judge: 1625
  • Author: SorahISA

Problem Description

PDF

Given a sequence of n numbers a1​,a2​,…,an​. Define maxℓ,r​ as max{aℓ​,aℓ+1​,…,ar​} and minℓ,r​ vise versa, please count the number of pairs (ℓ,r) such that 1≤ℓ≤r≤n and (maxℓ,r​−minℓ,r​)+(r−ℓ)=k.

  • 2≤n≤500000.
  • 0≤k≤n−2.
  • 1≤ai​≤n for i=1,2,…,n.

Chinese Editorial

10 points --- O(n2)

直接暴力兩個 for 迴圈就做完了，這是 送分 兼 確認題目用 的子任務。

int ans = 0;
for (int l = 0; l < n; ++l) {
    for (int r = l, mn = A[l], mx = A[l]; r < n; ++r) {
        mn = min(mn, A[r]), mx = max(mx, A[r]);
        if ((mx - mn) - (r - l) == k) ++ans;
    }
}
cout << ans << "\n";

65 ~ 80 points --- 突破 o(n2)

想像你現在想找到左界 ℓ 在 [L0​,L1​] 之間、右界 r 在 [R0​,R1​] 之間的所有區間的答案（[L0​,L1​] 跟 [R0​,R1​] 沒有交集）：

如果算式變成 r−ℓ=k 要怎麼做，能不能在 O(R1​−L0​) 做完？

作法

簡單嘛，枚舉 ℓ∈[L0​,L1​]，檢查 ℓ+k 有沒有落在 [R0​,R1​] 就好了～

如果算式變成 maxℓ,L1​​−(r−ℓ)=k 要怎麼做？

作法

先把式子跟 ℓ、r 有關的分開，得到

maxℓ,L1​​+ℓ−k=r

對一個 ℓ∈[L0​,L1​] 而言，定義 costL​(ℓ)=maxℓ,L1​​+ℓ，只要 costL​(ℓ)−k 落在 [R0​,R1​] 就有一組解。

而 costL​(ℓ) 可以透過紀錄 a 從 L1​ 往前的最大值來做到 O(1) 維護。

Sample Code 
int solve(int L0, int L1, int R0, int R1) {
    int ans = 0;
    for (int l = L1, mx = a[L1]; l >= L0; --l) {
        mx = max(mx, a[l]);
        if (R0 <= mx + l - k and mx + l - k <= R1) ++ans;
    }
    return ans;
}

如果算式變成 (maxℓ,L1​​−minR0​,r​)−(r−ℓ)=k 要怎麼做？

作法

一樣把式子整理到兩邊，變成

maxℓ,L1​​+ℓ−k=minR0​,r​+r

右式雖然 不再是一個區間，但我們還是能透過先計算出所有合法的右界 costR​(r):=minR0​,r​+r 之後，再枚舉 ℓ 來找到答案。

需要特別注意的是 costR​ 中可能會有 重複 的數字（e.g., [4,2,3,1] 的 costR​(1)=costR​(3)=costR​(4)=5），此時，我們也需要 多次 計算到這些右界帶來的貢獻。

以下的實作方式是使用 std::map 來記錄 costR​ 裡的所有數字跟出現次數，在從 L1​ 到 L0​ 遍歷 ℓ 時便可以直接計算答案。

Sample Code 
int solve(int L0, int L1, int R0, int R1) {
    int ans = 0;
    
    map<int, int> costR;
    for (int r = R0, mn = a[R0]; r <= R1; ++r) {
        mn = min(mn, a[r]);
        ++costR[mn + r];
    }
    
    for (int l = L1, mx = a[L1]; l >= L0; --l) {
        mx = max(mx, a[l]);
        ans += costR[mx + l - k];
    }
    
    return ans;
}

如果算式變成 max(maxℓ,L1​​,maxR0​,r​)−(r−ℓ)=k 要怎麼做？

作法

一樣把式子整理到兩邊......

咦？好像整理不了了耶 QwQ

那能不能試著 分開討論 可能會出現的 case 啊？

  1. maxℓ,L1​​>maxR0​,r​：最大值出現在 [ℓ,L1​] 裡面；
  2. maxℓ,L1​​<maxR0​,r​：最大值出現在 [R0​,r] 裡面；
  3. maxℓ,L1​​=maxR0​,r​：兩邊都有最大值！？
    • 還是先考慮 a 是 1,2,…,n 的排列好了......

(1.) 跟 (2.) 本質上是相同的，只是左右對調而已，以下說明只講解 (1.) 的部分。

如果我們希望讓最大值出現在 [ℓ,L1​]，那就代表此時的 r 不能太大，不然可能會把一個超過 maxℓ,L1​​ 的數字加進右半邊。

但這樣對不同的 ℓ 都會有不同的 r 範圍，要怎麼做到快速的維護呢？

考慮到當你把 ℓ 往左移一步的時候 maxℓ−1,L1​​=max(aℓ−1​,maxℓ,L1​​) 一定只會變大，所以隨著 ℓ 往 L0​ 移動，合法的 r 的 右界 一定也只會 往 R1​ 靠近（左界一定固定在 R0​）。

於是，我們可以使用 two-pointer 來維護：用兩個變數分別記錄當前的左右界 (ℓ,r)，並在每次移動 ℓ 的時候都使用 while 迴圈檢查 r 能往右移動到哪裡。

既然我們已經知道最大值出現在 [ℓ,L1​] 裡面，那我們就可以把式子整理成第二個提示中的算式：

maxℓ,L1​​+ℓ−k=r

剩下的實作部分請參考 code。

Sample Code 
int solve(int L0, int L1, int R0, int R1) {
    int ans = 0;
    
    for (int l = L1, r = R0-1, mx = a[L1]; l >= L0; --l) {
        mx = max(mx, a[l]);
        while (r < R1 and a[r+1] < mx) ++r;
        if (R0 <= mx + l - k and mx + l - k <= r) ++ans;
    }
    
    for (int l = L1+1, r = R0, mx = a[R0]; r <= R1; ++r) {
        mx = max(mx, a[r]);
        while (l > L0 and a[l-1] < mx) --l;
        if (l <= -mx + r + k and -mx + r + k <= L1) ++ans;
    }
    
    return ans;
}

如果算式變回原本那樣要怎麼做？

作法

先忽略 (L1​,R0​) 範圍的東西。也就是說，我們現在要找到所有 (ℓ,r) pair 使得

(max(maxℓ,L1​​,maxR0​,r​)−min(minℓ,L1​​,minR0​,r​))−(r−ℓ)=k

繼續分 case：

  1. maxℓ,L1​​>maxR0​,r​ 且 minℓ,L1​​<minR0​,r​：最大跟最小值都出現在 [ℓ,L1​] 裡面；
  2. maxℓ,L1​​<maxR0​,r​ 且 minℓ,L1​​>minR0​,r​：最大跟最小值都出現在 [R0​,r] 裡面；
  3. maxℓ,L1​​>maxR0​,r​ 且 minℓ,L1​​>minR0​,r​：最大值在左邊、最小值在右邊；
  4. maxℓ,L1​​<maxR0​,r​ 且 minℓ,L1​​<minR0​,r​：最大值在右邊、最小值在左邊。

因為對稱性，以下只分析 (1.) 跟 (3.) 的 case：

在 (1.) 最大最小值都出現在 [ℓ,L1​] 的時候，我們可以得出跟上一個提示一樣的性質：

  • 隨著 ℓ 往 L0​ 移動，合法的 r 的 右界 一定也只會 往 R1​ 靠近（左界也一定固定在 R0​）。

在 (3.) 最大值在左邊、最小值在右邊的時候，我們也可以得出類似的性質：

  • 隨著 ℓ 往 L0​ 移動，合法的 r 的 右界 一定也只會 往 R1​ 靠近。

但注意到 r 如果取太小的話，最小值可能就會跑到左邊去了，也就是說此時我們也會需要維護 r 的 左界：

  • 隨著 ℓ 往 L0​ 移動，minℓ,L1​​ 會越來越小，所以 r 的 左界 也要 往右移動 到 minR0​,r​ 足夠小的位置。

實際上，可以觀察到 (3.) 裡 r 的左界其實就是滿足 minR0​,r​<minℓ,L1​​ 的最小的 r（記 r3​），而又 (1.) 裡的右界就是滿足 minR0​,r​>minℓ,L1​​ 最大的 r（記 r1​）：

  • r1​,r3​ 必定滿足 r1​+1=r3​。

於是我們可以在移動 ℓ 時同時維護 r1​ 跟 r3​，將 (1.) 跟 (3.) 的 case 合併處理。

回到原本關鍵的問題：要怎麼將所有 (ℓ,r) pair（1≤ℓ≤r≤n）分進一些沒有交集的 (L0​,L1​,R0​,R1​) 裡面？

俗話說：「有兩個 index 就很欠分治」，就來嘗試看看吧！

可以從 Lab 2 Problem B (happily) 的做法得到靈感：

每次將區間 [L,R] 分成兩半 [L,M] 跟 [M+1,R] 遞迴處理，這樣

  1. 如果 (ℓ,r) 都落在左半邊或右半邊就會被遞迴處理到。
  2. 如果 ℓ 落在左半邊、r 落在右半邊，就會被當下的函式處理到。
int solve(int L, int R) {
    int M = (L + R) >> 1;
    int L0 = L, L1 = M, R0 = M+1, R1 = R;
    
    int ans = 0;
    /* calculate answer */
    return solve(L, M) + solve(M+1, R) + ans;
}

這也解釋了為什麼上面會說「先忽略 (L1​,R0​) 範圍的東西」，因為 L1​+1=R0​，這個範圍根本就沒有東西啊！

至此，我們已經有了一個 T(n)=2⋅T(n/2)+O(nlogn)⟹T(n)=O(nlog2n) 的做法了！

Sample Code for Subtask 1, 3, 4: O(nlog2n) 
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(0), cin.tie(0);
    
    int N, K; cin >> N >> K;
    
    vector<int> A(N);
    for (int &x : A) cin >> x;
    
    int64_t ans = (K == 0 ? N : 0);
    
    function<void(int, int)> recur = [&](int L, int R) {
        if (L == R) return;
        int M = (L + R) >> 1;
        recur(L, M), recur(M+1, R);
        
        static vector<int> mn(N), mx(N);
        static map<int, int> cnt1, cnt2;
        mn[M] = mx[M] = A[M];
        for (int i = M-1; i >= L; --i) mn[i] = min(mn[i+1], A[i]), mx[i] = max(mx[i+1], A[i]);
        mn[M+1] = mx[M+1] = A[M+1];
        for (int i = M+2; i <= R; ++i) mn[i] = min(mn[i-1], A[i]), mx[i] = max(mx[i-1], A[i]);
        
        for (int l = M, r1 = M, r2 = M; l >= L; --l) {
            /// r1: last pos which mn[r1] and mx[r1] is between mn[l] and mx[l] ///
            /// r1: [max - min - r + l = k] ==> [r = l - k + max - min] ///
            while (r1 < R and mn[r1+1] > mn[l] and mx[r1+1] < mx[l]) {
                ++r1;
                ++cnt1[r1];
                --cnt2[r1 + mn[r1]];
            }
            /// r2: last pos which mx[r2] is below mx[l] ///
            /// r2: [max - min - r + l = k] ==> [r + min = l - k + max] ///
            while (r2 < R and mx[r2+1] < mx[l]) {
                ++r2;
                ++cnt2[r2 + mn[r2]];
            }
            ans += cnt1[l - K + mx[l] - mn[l]];
            ans += cnt2[l - K + mx[l]];
        }
        cnt1.clear(), cnt2.clear();
        
        for (int l1 = M+1, l2 = M+1, r = M+1; r <= R; ++r) {
            /// l1: first pos which mn[l1] and mx[l1] is between mn[r] and mx[r] ///
            /// l1: [max - min - r + l = k] ==> [l = r + k + min - max] ///
            while (l1 > L and mn[l1-1] > mn[r] and mx[l1-1] < mx[r]) {
                --l1;
                ++cnt1[l1];
                --cnt2[l1 - mn[l1]];
            }
            /// l2: first pos which mx[l2] is below mx[r] ///
            /// l2: [max - min - r + l = k] ==> [l - min = r + k - max] ///
            while (l2 > L and mx[l2-1] < mx[r]) {
                --l2;
                ++cnt2[l2 - mn[l2]];
            }
            ans += cnt1[r + K + mn[r] - mx[r]];
            ans += cnt2[r + K - mx[r]];
        }
        cnt1.clear(), cnt2.clear();
    };
    
    recur(0, N-1);
    
    cout << ans << "\n";
    
    return 0;
}

或許你有這樣的想法：「把 std::map 換成 std::unordered_map 就直接是 O(nlogn) 了耶，一定會跑比較快吧！」，但其實 std::unordered_map 的 常數非常大，往往跑的比 std::map 還要更慢。

Or, do we?

對於任意相同的兩個數字 ai​,aj​，我們其實只需要把位置小的 在比較時 看成是 小一點點 的數字，就可以讓任兩個數字都不會被當成相同的值了！

  • 請試著想想看為什麼上面的計算不會爛掉。

更直接的，我們可以先將 a 以 (ai​,i) 重新編號 變成 a′，把 a′ 當成是 1,2,…,n 的排列，這樣就可以直接套用上面的做法了！

請仔細觀察下面 rk[] 的使用方式，mn[] 跟 mx[] 分別代表了什麼，以及思考為什麼在計算答案時要套上 A[]。

Sample Code for Subtask 1 ~ 5: O(nlog2n) 
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(0), cin.tie(0);
    
    int N, K; cin >> N >> K;
    
    vector<int> A(N);
    for (int &x : A) cin >> x;
    
    vector<pair<int, int>> A_sorted;
    vector<int> rk(N);
    for (int i = 0; i < N; ++i) A_sorted.emplace_back(A[i], i);
    sort(begin(A_sorted), end(A_sorted));
    for (int i = 0; i < N; ++i) rk[i] = lower_bound(begin(A_sorted), end(A_sorted), pair{A[i], i}) - begin(A_sorted);
    sort(begin(A), end(A)); /// use A[rk[i]] to find original A[i]
    
    int64_t ans = (K == 0 ? N : 0);
    
    function<void(int, int)> recur = [&](int L, int R) {
        if (L == R) return;
        int M = (L + R) >> 1;
        recur(L, M), recur(M+1, R);
        
        static vector<int> mn(N), mx(N);
        static map<int, int> cnt1, cnt2;
        mn[M] = mx[M] = rk[M];
        for (int i = M-1; i >= L; --i) mn[i] = min(mn[i+1], rk[i]), mx[i] = max(mx[i+1], rk[i]);
        mn[M+1] = mx[M+1] = rk[M+1];
        for (int i = M+2; i <= R; ++i) mn[i] = min(mn[i-1], rk[i]), mx[i] = max(mx[i-1], rk[i]);
        
        for (int l = M, r1 = M, r2 = M; l >= L; --l) {
            /// r1: last pos which mn[r1] and mx[r1] is between mn[l] and mx[l] ///
            /// r1: [max - min - r + l = k] ==> [r = l - k + max - min] ///
            while (r1 < R and mn[r1+1] > mn[l] and mx[r1+1] < mx[l]) {
                ++r1;
                ++cnt1[r1];
                --cnt2[r1 + A[mn[r1]]];
            }
            /// r2: last pos which mx[r2] is below mx[l] ///
            /// r2: [max - min - r + l = k] ==> [r + min = l - k + max] ///
            while (r2 < R and mx[r2+1] < mx[l]) {
                ++r2;
                ++cnt2[r2 + A[mn[r2]]];
            }
            ans += cnt1[l - K + A[mx[l]] - A[mn[l]]];
            ans += cnt2[l - K + A[mx[l]]];
        }
        cnt1.clear(), cnt2.clear();
        
        for (int l1 = M+1, l2 = M+1, r = M+1; r <= R; ++r) {
            /// l1: first pos which mn[l1] and mx[l1] is between mn[r] and mx[r] ///
            /// l1: [max - min - r + l = k] ==> [l = r + k + min - max] ///
            while (l1 > L and mn[l1-1] > mn[r] and mx[l1-1] < mx[r]) {
                --l1;
                ++cnt1[l1];
                --cnt2[l1 - A[mn[l1]]];
            }
            /// l2: first pos which mx[l2] is below mx[r] ///
            /// l2: [max - min - r + l = k] ==> [l - min = r + k - max] ///
            while (l2 > L and mx[l2-1] < mx[r]) {
                --l2;
                ++cnt2[l2 - A[mn[l2]]];
            }
            ans += cnt1[r + K + A[mn[r]] - A[mx[r]]];
            ans += cnt2[r + K - A[mx[r]]];
        }
        cnt1.clear(), cnt2.clear();
    };
    
    recur(0, N-1);
    
    cout << ans << "\n";
    
    return 0;
}

Imagine you want to find all intervals where the left boundary ℓ is in [L0​,L1​] and the right boundary r is in [R0​,R1​] for all non-overlapping intervals [L0​,L1​] and [R0​,R1​]:

If the equation becomes r−ℓ=k, how can you solve it in O(R1​−L0​) time?

Approach

It's simple, iterate over ℓ∈[L0​,L1​] and check if ℓ+k falls in [R0​,R1​].

If the equation becomes maxℓ,L1​​−(r−ℓ)=k, what can you do?

Approach

Separate the expression involving ℓ and r to obtain

maxℓ,L1​​+ℓ−k=r

For a given ℓ∈[L0​,L1​], define costL​(ℓ)=maxℓ,L1​​+ℓ. There is a solution if costL​(ℓ)−k falls in [R0​,R1​].

The value of costL​(ℓ) can be maintained in O(1) time by keeping track of the maximum value of a from L1​ backward.

Sample Code 
int solve(int L0, int L1, int R0, int R1) {
    int ans = 0;
    for (int l = L1, mx = a[L1]; l >= L0; --l) {
        mx = max(mx, a[l]);
        if (R0 <= mx + l - k and mx + l - k <= R1) ++ans;
    }
    return ans;
}

If the equation becomes (maxℓ,L1​​−minR0​,r​)−(r−ℓ)=k, how can you solve it?

Approach

Similarly, rearrange the equation to

maxℓ,L1​​+ℓ−k=minR0​,r​+r

Even though the right side is no longer an interval, we can calculate all valid right boundaries costR​(r):=minR0​,r​+r first. Then, iterate over ℓ to find the solution.

Note that costR​ may contain duplicates (e.g., [4,2,3,1] with costR​(1)=costR​(3)=costR​(4)=5), so we need to calculate the contribution of these right boundaries multiple times.

The following implementation uses std::map to record all numbers and their occurrences in costR​. While traversing ℓ from L1​ to L0​, we can directly calculate the answer.

Sample Code 
int solve(int L0, int L1, int R0, int R1) {
    int ans = 0;
    
    map<int, int> costR;
    for (int r = R0, mn = a[R0]; r <= R1; ++r) {
        mn = min(mn, a[r]);
        ++costR[mn + r];
    }
    
    for (int l = L1, mx = a[L1]; l >= L0; --l) {
        mx = max(mx, a[l]);
        ans += costR[mx + l - k];
    }
    
    return ans;
}

If the equation becomes max(maxℓ,L1​​,maxR0​,r​)−(r−ℓ)=k, what can you do?

Approach

Similarly, rearrange the equation...

Hmm? It seems like it can't be rearranged, huh? QwQ

Can we try to discuss separately the possible cases that might arise?

  1. maxℓ,L1​​>maxR0​,r​: The maximum value is in [ℓ,L1​];
  2. maxℓ,L1​​<maxR0​,r​: The maximum value is in [R0​,r];
  3. maxℓ,L1​​=maxR0​,r​: Both sides have the maximum value!?
    • Maybe let's first consider a is a permutation of 1,2,…,n...

(1.) and (2.) are essentially the same, just with left and right swapped. The explanation below focuses on (1.).

If we want the maximum value to appear in [ℓ,L1​], it means that at this time, r should not be too large, or else we may include a number greater than maxℓ,L1​​ in the right half.

But how can we quickly maintain different ranges for r as ℓ moves left?

Considering that when you move ℓ one step to the left, maxℓ−1,L1​​=max(aℓ−1​,maxℓ,L1​​) will definitely only increase. Therefore, as ℓ moves towards L0​, the right boundary of valid r will only move closer to R1​ (the left boundary will always be fixed at R0​).

Thus, we can use a two-pointer approach to maintain: use two variables to record the current left and right boundaries (ℓ,r) and use a while loop to check where r can move to each time ℓ moves.

Now that we know the maximum value is in [ℓ,L1​], we can rearrange the equation into the same form as the second hint:

maxℓ,L1​​+ℓ−k=r

Sample Code

If the equation returns to its original form, what can you do?

Approach

First, ignore the range (L1​,R0​). In other words, we want to find all (ℓ,r) pairs such that

(max(maxℓ,L1​​,maxR0​,r​)−min(minℓ,L1​​,minR0​,r​))−(r−ℓ)=k

Continue to split the cases:

  1. maxℓ,L1​​>maxR0​,r​ and minℓ,L1​​<minR0​,r​: Both maximum and minimum values are in [ℓ,L1​];
  2. maxℓ,L1​​<maxR0​,r​ and minℓ,L1​​>minR0​,r​: Both maximum and minimum values are in [R0​,r];
  3. maxℓ,L1​​>maxR0​,r​ and minℓ,L1​​>minR0​,r​: Maximum value is on the left, minimum value is on the right;
  4. maxℓ,L1​​<maxR0​,r​ and minℓ,L1​​<minR0​,r​: Maximum value is on the right, minimum value is on the left.

Due to symmetry, the analysis below only discusses cases (1.) and (3.):

In case (1.) where both maximum and minimum values are in [ℓ,L1​], we can obtain a similar property as in the previous hint:

  • As ℓ moves towards L0​, the right boundary of valid r will only move closer to R1​ (the left boundary will always be fixed at R0​).

In case (3.) where the maximum value is on the left and the minimum value is on the right, we can also deduce a similar property:

  • As ℓ moves towards L0​, the right boundary of valid r will only move closer to R1​.

However, note that if r becomes too small, the minimum value may end up on the left. Therefore, we also need to maintain the left boundary of r as ℓ moves towards L0​:

  • As ℓ moves towards L0​, minℓ,L1​​ will become smaller, so the left boundary of r must also move to the right to a small enough position relative to minR0​,r​.

In practice, it can be observed that in case (3.), the left boundary of r is actually the smallest r that satisfies minR0​,r​<minℓ,L1​​ (denoted as r3​), and in case (1.), the right boundary is the largest r that satisfies minR0​,r​>minℓ,L1​​ (denoted as r1​):

  • r1​,r3​ must satisfy r1​+1=r3​.

Therefore, we can maintain r1​ and r3​ simultaneously while moving ℓ, and reset them to r1​=r3​ after finishing Case 1 and Case 3. During the movement, decrement cnt1[] and cnt2[] in reverse order.

  • Think about why we need to move r2 before moving r1.

Now we have a solution with T(n)=2⋅T(n/2)+O(nlogn)⟹T(n)=O(nlog2n)!

Sample Code for Subtask 1, 3, 4: O(nlog2n) 
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(0), cin.tie(0);
    
    int N, K; cin >> N >> K;
    
    vector<int> A(N);
    for (int &x : A) cin >> x;
    
    int64_t ans = (K == 0 ? N : 0);
    
    function<void(int, int)> recur = [&](int L, int R) {
        if (L == R) return;
        int M = (L + R) >> 1;
        recur(L, M), recur(M+1, R);
        
        static vector<int> mn(N), mx(N);
        static map<int, int> cnt1, cnt2;
        mn[M] = mx[M] = A[M];
        for (int i = M-1; i >= L; --i) mn[i] = min(mn[i+1], A[i]), mx[i] = max(mx[i+1], A[i]);
        mn[M+1] = mx[M+1] = A[M+1];
        for (int i = M+2; i <= R; ++i) mn[i] = min(mn[i-1], A[i]), mx[i] = max(mx[i-1], A[i]);
        
        for (int l = M, r1 = M, r2 = M; l >= L; --l) {
            /// r1: last pos which mn[r1] and mx[r1] is between mn[l] and mx[l] ///
            /// r1: [max - min - r + l = k] ==> [r = l - k + max - min] ///
            while (r1 < R and mn[r1+1] > mn[l] and mx[r1+1] < mx[l]) {
                ++r1;
                ++cnt1[r1];
                --cnt2[r1 + mn[r1]];
            }
            /// r2: last pos which mx[r2] is below mx[l] ///
            /// r2: [max - min - r + l = k] ==> [r + min = l - k + max] ///
            while (r2 < R and mx[r2+1] < mx[l]) {
                ++r2;
                ++cnt2[r2 + mn[r2]];
            }
            ans += cnt1[l - K + mx[l] - mn[l]];
            ans += cnt2[l - K + mx[l]];
        }
        cnt1.clear(), cnt2.clear();
        
        for (int l1 = M+1, l2 = M+1, r = M+1; r <= R; ++r) {
            /// l1: first pos which mn[l1] and mx[l1] is between mn[r] and mx[r] ///
            /// l1: [max - min - r + l = k] ==> [l = r + k + min - max] ///
            while (l1 > L and mn[l1-1] > mn[r] and mx[l1-1] < mx[r]) {
                --l1;
                ++cnt1[l1];
                --cnt2[l1 - mn[l1]];
            }
            /// l2: first pos which mx[l2] is below mx[r] ///
            /// l2: [max - min - r + l = k] ==> [l - min = r + k - max] ///
            while (l2 > L and mx[l2-1] < mx[r]) {
                --l2;
                ++cnt2[l2 - mn[l2]];
            }
            ans += cnt1[r + K + mn[r] - mx[r]];
            ans += cnt2[r + K - mx[r]];
        }
        cnt1.clear(), cnt2.clear();
    };
    
    recur(0, N-1);
    
    cout << ans << "\n";
    
    return 0;
}

Full Score Solution

Although using std::map is convenient and many issues like "how big should the array be", "will it go out of bounds", and "how to zero out the array" are not a concern, it still adds an additional O(logn) complexity. So, let's try to replace it with a better data structure.

  • No, it's not using std::unordered_map.
Observe how much we poke the std::map
  • r1: Roughly [0,n]
  • r1 + mn[r1]: Roughly [0,2n]
  • r2 + mn[r2]: Roughly [0,2n]
  • l - K + mx[l] - mn[l]: Roughly [−n,2n] (because mx[l] ≥ mn[l])
  • l - K + mx[l]: Roughly [−n,2n]
  • l1: Roughly [0,n]
  • l1 - mn[l1]: Roughly [−n,n]
  • l2 - mn[l2]: Roughly [−n,n]
  • r + K + mn[r] - mx[r]: Roughly [−n,2n] (because mx[r] ≥ mn[r])
  • r + K - mx[r]: Roughly [−n,2n]

Actually, we only need to access elements within the range of [−n,2n]. Therefore, we can replace std::map with a std::vector of length 3n+1. To access elements, we can simply shift by n and use vec[x + n].

We have addressed the issues of "how large should the array be" and "will it go out of bounds." However, we still need to solve the problem of zeroing out the array. We cannot directly use a for loop to zero out the array, and methods like std::fill or memset are not suitable since they take O(n) time to zero out the entire array, not O(interval length).

We can utilize a technique known as backtracking through time.

Consider the calculation of Cases 1 and 3 in the first half:

  • We know that cnt1[] only affects the position of r.
  • We know that cnt2[] only affects the position of r + A[mn[r]].
  • We also know the final values of r1 and r2.

After calculating Cases 1 and 3, we can move r2 back to r1 and then move r1 back to the position M. During these movements, we can decrement the values in cnt1[] and cnt2[] in reverse. This process allows us to clear an interval in O(R−L) time!

Think about why we need to move r2 before r1.

  • Time complexity: T(n)=2⋅T(n/2)+O(n)⟹T(n)=O(nlogn).
  • Space complexity: O(n).

int gogo(int L, int R, int K) {
    if (L == R-1) return (K == 0);
    int M = (L + R) >> 1;

    for (int i = M - 1; i >= L; --i) {
        suf_mn[i] = min(suf_mn[i+1], A[i]);
        suf_mx[i] = max(suf_mx[i+1], A[i]);
    }
    for (int i = M; i < R; ++i) {
        pre_mn[i] = min(pre_mn[i-1], A[i]);
        pre_mx[i] = max(pre_mx[i-1], A[i]);
    }

    int ret = 0;
    int R_min = M, R_max = M;
    int mx = -1, mn = INF;
    for (int i = M - 1; i >= L; --i) {
        chmax(mx, A[i]), chmin(mn, A[i]);
        while (R_max < R && pre_mx[R_max] <= mx) {
            cnt[pre_mn[R_max] + R_max]++;
            R_max++;
        }
        while (R_min < R_max && pre_mn[R_min] >= mn) {
            cnt[pre_mn[R_min] + R_min]--;
            R_min++;
        }
        int R_target = mx - mn + i - K;
        ret += (M <= R_target && R_target < R_min);
        if (mx + i - K >= 0) ret += max(int(0), cnt[mx + i - K]);
    }
    while (R_min < R_max) cnt[pre_mn[R_min] + R_min]--, R_min++;

    int L_min = M-1, L_max = M-1;
    mx = -1, mn = INF;
    for (int i = M; i < R; ++i) {
        chmax(mx, A[i]), chmin(mn, A[i]);
        while (L_max >= L && suf_mx[L_max] < mx) {
            cnt1[suf_mn[L_max] - L_max + maxn]++;
            L_max--;
        }
        while (L_min > L_max && suf_mn[L_min] >= mn) {
            cnt1[suf_mn[L_min] - L_min + maxn]--;
            L_min--;
        }
        int tg = mn - mx + i + K;
        ret += (L_min < tg && tg < M);
        if (mx - i - K + maxn >= 0) ret += max(int(0), cnt1[mx - i - K + maxn]);
    }
    while (L_min > L_max) cnt1[suf_mn[L_min] - L_min + maxn]--, L_min--;

    for (int i = M - 1; i >= L; --i) {
        suf_mn[i] = INF, suf_mx[i] = -1;
    }
    for (int i = M; i < R; ++i) {
        pre_mn[i] = INF, pre_mx[i] = -1;
    }
    return ret + gogo(L, M, K) + gogo(M, R, K);
}
