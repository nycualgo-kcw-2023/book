C. Pecola or Treat

Problem Description

PDF

一個賽程 (Schedule) 裡面包含多個決鬥 (Battle)，每次決鬥最多只能有 k 人參加，且有一位會進行之後的賽程。

定義一個賽程的成本 c(S′)=∑dS′​[i]×a[i]，其中 dS′​[i] 為第 i 參賽者在賽程 S′ 中，最多需要參加的決鬥數量。

你需要找到一個賽程 S ，使得 c(S) 是最小的。

輸出 c(S) 以及描述該賽程。

根據以上敘述，可以發現一個賽程等同於 k 元樹，每個參賽者可以視為葉子節點，而 dS​[i] 則是每個葉子節點的深度。

A schedule contains multiple battles, and each battle can have a maximum of k participants. Additionally, there is a subsequent schedule for the winner of each battle.

Define the cost of a schedule c(S′)=∑dS′​[i]×a[i], where dS′​[i] is the maximum number of battles participant i needs to attend in schedule S′.

Your task is to find a schedule S such that c(S) is minimized.

Output c(S) and describe the corresponding schedule.

Based on the description above, a schedule can be considered equivalent to a k-ary tree, where each participant is a leaf node, and dS​[i] represents the depth of each leaf node.

Chinese Editorial

Subtask 1

當 n=1 時，不需要進行決鬥就能決定出勝者， c(S)=0。

其餘狀況，因為 n≤k，所有參賽者都可以在一次決鬥中就決定勝者，且至少都需要進行一次決鬥，所以最小的 c(S)=∑ai​。

Subtask 2

仔細觀察題目的敘述，可以將 a[i] 視作 Huffman encoding 中的頻率，dS​[i] 視作編碼的長度。

那麼 k=2 就完全等同於有 n 個字元的 Huffman Encoding 問題。

因此，我們可以像講義中所提到的，將節點中頻率最小的兩個節點 x,y 挑出並組成一個新的節點 z，且其頻率為 x+y。不斷重複，直到節點數 ≤1 。

使用 priority queue 去找到並移除過程中最小頻率的兩個節點 x,y，再將新節點 z 放進去。這過程就等同題目的，挑選 x,y 進行決鬥，決鬥的勝者作為 z 進行剩下的賽程。

Subtask 3 & 4

接著將題目擴展到 k>2，一個直覺的想法是用跟 Subtask 2 一樣的操作，每次從 priority queue 裡面挑出 k 個權重最小的節點，將這些節點的總和作為新節點 z 的權重，最後放回 priority queue。

然而這樣做真的是最好的嗎？套用到 Sample Case 2，會發現得到的答案不是最佳解。 不過稍微觀察，會發現在 n≡1mod(k−1) 的時候 (Subtask 3)，直接挑權重前 k 小的能保證有最佳解。

  • Claim 1: 所有深度不是最深的節點，其 Sibling 節點的數量必須為 k，且子節點要有兩個以上

  • Claim 2: 對於任意葉子節點 i,j，若a[i]≤a[j]，則 dS​[i]≥dS​[j]

    類似地，如果不是的話，可以將兩葉子節點的位置交換，能使得 c(S) 更小。

  • Claim 3: 對於任意葉子節點，若將 i 移到同深度的其他位置，答案不變

    因為深度一樣，所以任意交換不會影響 c(S) 的結果

Greedy Algorithm A 
A(a, k) // a is set of node()
	S := a
	cost := 0
	while |S| ≠ 1 :
			Z := new node()
			Z.childs := ∅ // set of node()
			Z.val := 0
			while |Z.childs| < 2 or |S| % (k - 1) != 0 : // at least 2 in one battle

					S := S - minNode
			cost += Z.val
			S := S ∪ Z
	root := the remained node in S
	return root, cost

English Editorial

Subtask 1

When n=1, no battles are needed to determine the winner, and c(S)=0.

For other cases where n≤k, all participants can determine the winner in a single battle. At least one battle is required, so the minimum c(S)=∑ai​.

Subtask 2

By carefully observing the problem description, a[i] can be regarded as frequencies in Huffman encoding, and dS​[i] can be viewed as the length of the encoding.

For k=2, this is equivalent to the Huffman Encoding problem with n characters.

Therefore, as mentioned in the lecture notes, we can select the two nodes with the smallest frequencies, x and y, and create a new node z with a frequency of x+y. This process is repeated until the number of nodes is less than or equal to 1.

Use a priority queue to find and remove the two nodes with the smallest frequencies, x and y, and then insert the new node z. This process is equivalent to selecting x and y for a battle, and the winner continues in the remaining schedule.

Subtasks 3 & 4

Next, extend the problem to k>2. An intuitive approach is to perform similar operations as in Subtask 2. Each time, select k nodes with the smallest weights from the priority queue, use their sum as the weight of the new node z, and finally, insert it back into the priority queue.

However, is this truly the best approach? Applying it to Sample Case 2, we find that the obtained answer is not optimal. Yet, with a closer observation, it is noted that when n≡1mod(k−1) (Subtask 3), selecting the smallest k weights directly ensures the optimal solution.

Continuing with a more in-depth analysis, for the optimal schedule S (k-ary tree), we have the following claims:

  • Claim 1: For all non-deepest nodes, the number of sibling nodes must be k, and each node must have at least two child nodes.

    It can be observed that if this is not the case, moving a deeper node up can reduce c(S).

  • Claim 2: For any leaf nodes i and j, if a[i]≤a[j], then dS​[i]≥dS​[j].

    Similarly, if not, swapping the positions of the two leaf nodes can reduce c(S).

  • Claim 3: For any leaf node, moving it to another position at the same depth does not change the answer.

    Since the depth is the same, any exchange does not affect the result of c(S).

Based on the observations above, let's attempt to infer the following Greedy algorithm and provide a proof of its correctness.

Greedy Algorithm A 
A(a, k) // a is set of node()
	S := a
	cost := 0
	while |S| ≠ 1 :
			Z := new node()
			Z.childs := ∅ // set of node()
			Z.val := 0
			while |Z.childs| < 2 or |S| % (k - 1) != 0 : // at least 2 in one battle
					minNode := node with smallest val in S
					Z.childs = Z.childs ∪ minNode
					Z.val += minNode.val
					S := S - minNode
			cost += Z.val
			S := S ∪ Z
	root := the remained node in S
	return root, cost

To prove that for a fixed k, for any n participants, the schedule produced by algorithm A is optimal:

Let Hn​ be the schedule produced by A, and Tn​ be the optimal schedule. We want to prove that c(Hn​)≤c(Tn​)∀a,k.

We can use induction:

  • Base: 1≤n≤k:
    • For n=1, no competition is needed.
    • For 2≤n≤k, only one competition is needed.
    • In both cases, A can obviously produce the best schedule.
  • Induction: ∀n<l:c(Hn​)≤c(Tn​)⇒c(Hl​)≤c(Tl​):
    • For any combination of participants where n<l, we have c(Hl​)≤c(Tl​).
    • Prove by contradiction: Suppose there exists Tl​:c(Hl​)>c(Tl​).

      • According to Claim 1, Claim 2, and Claim 3, we can transform Tl​ into a new schedule Tl∗​, where the k′ smallest-weight participants participate in a battle, and k′=k−[(n−1)mod(k−1)]. Additionally, c(Tl​)≥c(Tl∗​).

        Let y be the set of the k′ smallest weights.

        Let z′ be the sum of the k′ smallest weights.

        Let a′:=a∪z′−y, ∣a′∣=l−k′+1.

        After transforming to schedule Tl∗​, we can guarantee that Tl∗​ will have a battle composed of participants from y. Similarly, according to A, the schedule Hl​ will also have a battle composed of participants from y.

        We remove the battle composed of y in Tl∗​, resulting in Tl−k′+1∗​. This provides a solution satisfying a′.

        By directly using A to produce the schedule for a′, denoted as Hl−k′+1​, we know that c(Hl​)=c(Hl−k′+1​)+z′;c(Tl∗​)=c(Tl−k′+1∗​)+z′, and because c(Hl​)>c(Tl​)≥c(Tl∗​), we can deduce that c(Hl−k′+1​)>c(Tl−k′+1∗​).

        This contradicts our assumption ∀n<l:c(Hn​)≤c(Tn​).

#include <iostream>
#include <vector>
#include <queue>

#define rep(i, a, b) for (int i = a; i < b; i++)
#define rrep(i, a, b) for (int i = a; i <= b; i++)
#define val first
#define idx second

using namespace std;
using ll = long long;
using pli = pair<ll, int>;

int main()
{
	cin.tie(0);
	ios_base::sync_with_stdio(0);

	int n, k, rating;
	priority_queue<pli, vector<pli>, greater<pli>> pq;
	/**************** input ****************/
	cin >> n >> k;
	for(int i = 1; i <= n; i++)
	{
		cin >> rating;
		pq.push({rating, i});
	}

	/**************** build schedule ****************/
	vector<vector<int>> battles;
	ll ans = 0;
	int nodeIdx = n;
	while (pq.size() != 1) // until only one member left
	{
		ll sum = 0;												 // value of the new battle
		battles.push_back({});									 // new battle
		while (battles.back().size() < 2 || pq.size() % (k - 1)) // put the k' smallest nodes into battles
		{
			pli node = pq.top();
			pq.pop();
			sum += node.val;
			battles.back().push_back(node.idx);
		}
		ans += sum;
		pq.push({sum, ++nodeIdx});
	}

	/**************** output ****************/
	cout << ans << '\n';
	cout << battles.size() << '\n';
	for(int i = 0; i < battles.size(); i++)
	{
		cout << battles[i].size() << ' ';
		for(int j = 0; j < battles[i].size(); j++)
		{
			cout << battles[i][j] << " \n"[j == battles[i].size() - 1];
		}
	}
	return 0;
}
