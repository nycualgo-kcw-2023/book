515505 - Introduction to Algorithms

I. Classical Data Structure Problem

  • Online Judge: 1608
  • Author: SorahISA

Problem Description

PDF

給定一個集合 S={a[0],a[1],…,a[n−1]}，你需要支援對這個集合進行 q 次操作，這些操作分為三種類型：

  1. 從 S 中移除最小的整數。保證 S=∅。
  2. 從 S 中移除最大的整數。保證 S=∅。
  3. 將 MEX(S) 插入 S 中。

這裡的 MEX(S) 被定義為在 S 中尚未出現的最小非負整數。

令 ans[i] 為第 ith 次移除或插入的整數，請輸出 ans[0],ans[1], …,ans[q−1]。

You are given a set S={a[0],a[1],…,a[n−1]}, and you will have to support q operations of three types on this set:

  1. Remove the smallest integer from S. It is guaranteed that S=∅.
  2. Remove the largest integer from S. It is guaranteed that S=∅.
  3. Insert MEX(S) into S.

Define MEX(S) as the smallest non-negative integer that has not appeared in S.

Let ans[i] be the ith removed or inserted integer, please output ans[0],ans[1], …,ans[q−1].

Chinese Editorial

子任務 1

可以直接將整個 Set 視為排序後的陣列。

最小最大值就是 a0​ 跟 an−1​，都可以在 O(n)（用 std::vector 的 erase）到 O(1)（用 std::deque 的 pop_front、 pop_back ）的時間内將它們刪除。

陣列的 MEX 的求法就是對 i=0,1,…,n−1 檢査：如果 ai​=i 就代表陣列中不存在 i。

要特別注意可能整個陣列包含了 0,1,…,n−1 的所有數字，這時的 MEX 値就是 n。

以下是求 MEX 的 sample code:

Sample code of MEX 
int mex(vector<int> &vec) {
	// vec is sorted.
	// all elements in vec is non-negative and unique
	int N = vec.size();
	for (int i = 0; i < N; ++i) {
		if (vec[i] != i) return i;
	}
	return N;
}

子任務 2

沒有會加數字的操作 3，只有不斷刪除最小値跟最大値的操作。 那麼就直接開一個 deque 先把數字都排序好，再依據操作來刪掉前後端的數字即可。

子任務 3

只有刪除最大値跟加入 MEX 兩種操作。

  • Case 1: 如果序列裡的數字是從零開始的連續非負整數們，那刪除 max 跟加入 MEX 都會變得很單純，只會動到序列的最後一個數字而已。
    • max{0,1,2,…,k}=k
    • MEX{0,1,2,…,k}=k+1
  • Case 2: 如果序列裡的數字還不是從零開始的連續非負整數們（有洞），那刪除 max 跟加入 MEX 的數字們就還不會有交集。
    • 證明很單純：假設 k 是最大的 <an−1​ 且沒出現在 a 裡面的數字。那刪除 max 會去把 an−1​ 刪掉，而加入 MEX 會把一個 ≤k<an−1​ 的數字加進去（補洞）。
    • 於是，可以維護一個變數紀錄上一次的 MEX 值是多少：
      • 只要這個值還小於序列 a 的長度，就代表還有洞要補，而這次的 MEX 也一定會比上一次的還要大。
      • 當這個值等於序列 a 的長度時就會變成 Case 1。

子任務 4

現在多了刪最小値的操作，相當於是在後院鑿洞，上個子任務存在的 MEX 遞增的性質也被破壞掉了。不如直接換個思路？

把相鄰的數字「黏」在一起當成一個區間，這樣在算 MEX 的時候就可以一次跳過一整段數字。

因為跳過一段數字就會掉進一個洞裡，所以其實跳一次就會直接得到答案了！

刪最小値、刪最大値的操作也仍然只是修改第一個或最後一個區間的端點。

舉個例子：假設在這個子任務依序進行以下操作，那麼會長這樣：

  1. Empty
  2. (Insert MEX(S)) →[0,0]
  3. (Insert MEX(S)) →[0,1]
  4. (Insert MEX(S)) →[0,2]
  5. (Insert MEX(S)) →[0,3]
  6. (Insert MEX(S)) →[0,4]
  7. (Remove min(S)) →[1,4]
  8. (Remove min(S)) →[2,4]
  9. (Remove min(S)) →[3,4]
  10. (Insert MEX(S)) →[0,0],[3,4]
  11. (Remove max(S)) →[0,0],[3,3]
  12. (Remove min(S)) →[3,3]
  13. (Insert MEX(S)) →[0,0],[3,3]
  14. (Insert MEX(S)) →[0,1],[3,3]
  15. (Insert MEX(S)) →[0,3]

如果在刪除時把區間整個刪除了就移除、查詢 MEX 就是看第一個區間包不包含 0：

  • 包含 0 時 MEX 就會是該區間的右界 +1。
  • 不包含 0 時 MEX 就會是 0。

實作上需要注意當相鄰兩個區間接觸到彼此就需要將其合併，而這只會發生在前兩個區間。

子任務 5

子任務 4 其實已經把整個解法講完了。

這裡 N=0，代表一開始已經有一些區間們在裡面了。

只要好好跟他們打招呼就可以 AC 了！

子任務 5 的實作

使用 deque 包 pair 維護區間左右界。

記得要對陣列排序過，並好好打招呼。

更新時也要記得讓鄰居住一起。

結論

要會分析算法的複雜度、以及觀察題目的性質！

在本題中的性質就是 MEX 只會出現在從 0 開始的連續一段數字之後，所以加速的做法就是把連續的數字黏起來，做到 O(1) 査詢。

對 std::vector 做 erase 的複雜度不是 O(1)，他會需要把後面的元素一個一個往前搬。

English Editorial

Subtask 1

The entire set can be treated as a sorted array.

The minimum and maximum values are a0​ and an−1​, and they can be deleted in O(n) time (using std::vector's erase) to O(1) time (using std::deque's pop_front, pop_back).

The calculation of MEX for the array involves checking for i=0,1,…,n−1: if ai​=i, it means that i does not exist in the array.

It is essential to note that if the entire array contains all numbers from 0 to n−1, the MEX value is n.

Below is a sample code for calculating MEX:

Sample code for MEX 
int mex(vector<int> &vec) {
	// vec is sorted.
	// all elements in vec is non-negative and unique
	int N = vec.size();
	for (int i = 0; i < N; ++i) {
		if (vec[i] != i) return i;
	}
	return N;
}

Subtask 2

There are no operations to add numbers; only operations to continuously delete the minimum and maximum values. In this case, create a deque, sort the numbers, and delete the front and back numbers based on the operations.

Subtask 3

There are only two operations: delete the maximum value and add MEX.

  • Case 1: If the numbers in the sequence are consecutive non-negative integers starting from zero, deleting max and adding MEX become simple, affecting only the last number in the sequence.
    • max{0,1,2,…,k}=k
    • MEX{0,1,2,…,k}=k+1
  • Case 2: If the sequence contains non-consecutive non-negative integers (has gaps), numbers of deleting max and adding of MEX will not intersect.
    • The proof is simple: assuming k is the largest number not in a and <an−1​. Deleting max removes an−1​, and adding MEX inserts a number ≤k<an−1​ (filling the gap).
    • Therefore, maintain a variable to record the last MEX value:
      • As long as this value is less than the length of the sequence a, there are still gaps to fill, and the added MEX will be greater than the previous one.
      • When this value equals the length of the sequence a, it becomes Case 1.

Subtask 4

Now, the operation to delete the minimum value is added, disrupting the increasing property of MEX observed in the previous subtask. How about changing the approach?

Combine adjacent numbers into an "interval" so that when calculating MEX, an entire segment of numbers can be skipped at once.

Since skipping a segment means falling into a gap, one jump gives the answer directly!

Operations to delete the minimum and maximum values still only modify the endpoints of the first or last interval.

For example: if the following operations are performed in this subtask:

  1. Empty
  2. (Insert MEX(S)) →[0,0]
  3. (Insert MEX(S)) →[0,1]
  4. (Insert MEX(S)) →[0,2]
  5. (Insert MEX(S)) →[0,3]
  6. (Insert MEX(S)) →[0,4]
  7. (Remove min(S)) →[1,4]
  8. (Remove min(S)) →[2,4]
  9. (Remove min(S)) →[3,4]
  10. (Insert MEX(S)) →[0,0],[3,4]
  11. (Remove max(S)) →[0,0],[3,3]
  12. (Remove min(S)) →[3,3]
  13. (Insert MEX(S)) →[0,0],[3,3]
  14. (Insert MEX(S)) →[0,1],[3,3]
  15. (Insert MEX(S)) →[0,3]

If the interval becomes empty then just remove it from our consideration.

Querying MEX is whether the first interval contains 0:

  • If it contains 0, MEX will be the right boundary of that interval +1.
  • If it does not contain 0, MEX will be 0.

It is essential to merge adjacent intervals when they touch each other, which only occurs between the first two intervals.

Subtask 5

Subtask 4 has essentially covered the entire solution.

Here, N=0, indicating that some intervals are already present initially.

Just greet them properly, and you will get an AC!

Implementation for Subtask 5

Use a deque to store pairs representing intervals.

Make sure to sort the array and greet the existing intervals properly.

Also, remember to let the neighbors live together during updates.

Conclusion

It's crucial to analyze the algorithm's complexity and observe the properties of the problem!

In this problem, the property is that MEX only appears after a continuous segment of numbers starting from 0, so the accelerated approach is to glue consecutive numbers together, achieving O(1) queries.

Note that the complexity of using erase on std::vector is not O(1); it requires shifting the elements one by one.

#include "1608.h"
#include <bits/stdc++.h>
using namespace std;

deque<pair<int, int>> itv;

void init(int n, vector<int> a) {
    sort(begin(a), end(a));
    for (int x : a) {
        if (!itv.empty() and x == itv.back().second + 1) ++itv.back().second;
        else itv.emplace_back(x, x);
    }
}

int remove_min() {
    int ans = itv[0].first++;
    if (itv[0].first > itv[0].second) itv.pop_front();
    return ans;
}

int remove_max() {
    int ans = itv.back().second--;
    if (itv.back().first > itv.back().second) itv.pop_back();
    return ans;
}

int insert_mex() {
    if (itv.empty() or itv[0].first > 1) {
        itv.emplace_front(0, 0);
        return 0;
    }
    else if (itv[0].first == 1) {
        itv[0].first = 0;
        return 0;
    }
    else {
        int ans = ++itv[0].second;
        if ((int)itv.size() >= 2 and itv[0].second + 1 == itv[1].first) itv.pop_front(), itv[0].first = 0;
        return ans;
    }
}
