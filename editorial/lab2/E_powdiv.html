Chinese Editorial

k=1 --- Binary Exponentiation

課堂上有教過如何在 O(logy) 的時間內計算 xymodp，這題在 k=1 時，可以直接套用這個方法。

把這題的標題「Binary Exponentiation」放到 Google 搜尋也能找到許多相關的教學文章。

快速冪的具體概念就是將 f(y):=xy 拆成 重複的子問題 遞迴解決，觀察到 f(y)=f(a)⋅f(y−a)：

f(y)={f(⌊y/2⌋)⋅f(⌊y/2⌋)f(⌊y/2⌋)⋅f(⌊y/2⌋)⋅f(1)​if y is even,if y is odd.​

顯然不需要遞迴兩次重複計算一樣的 f(⌊y/2⌋)，所以可以先 將其記錄下來 以便直接取用。

快速冪－遞迴版本 
int fpow(int base, int exp, int mod) {
    if (exp == 0) return 1 % mod;
    int half = fpow(base, exp/2, mod);
    // return half * half % mod * (exp & 1 ? base : 1) % mod;
    if (exp % 2 == 0)
        return half * half % mod;
    else /// remember to take mod to reduce the maximum value during calculation ///
        return half * half % mod * base % mod;
    /*                     ^^^^^                                                  */
}

以下再介紹一個不依賴遞迴的實作方式。

這個實作方式利用到了 y 的 二進位表示法 ∑i=0∞​2ibi​，其中 bi​∈{0,1}，這樣就可以將 f(y) 拆成 ∏i=0∞​f(2ibi​)。

在 C++ 可以使用 y & (1 << i) 來取得 bi​，而 y >> i 可以取得 ⌊y/2i⌋。以下範例程式碼的實作方式是不斷將次方項用 >>= 1 除以二之後以 & 1 來取出「個位數」的值。

快速冪－二進制版本 
int fpow(int base, int exp, int mod) {
    int ans = 1 % mod;
    while (exp) {
        if (exp & 1) ans = ans * base % mod;
        exp >>= 1, base = base * base % mod;
    }
    return ans;
}

滿分解 1 --- 將 x 表達成 ak+q 的形式

要達到滿分，需要得出一個重要的觀察：

k 的倍數部分被模 p 也沒差，但如果餘數部分被模 p，就會無法產生正確的貢獻。

不妨將待相乘的數字表達成 ⟨a,q⟩，代表 ak+q，其中 0≤q<k。觀察兩組數字 ⟨a,q⟩ 跟 ⟨b,r⟩ 相乘的結果：

(ak+q)(bk+r)=abk2+(ar+bq)k+qr=(abk+ar+bq)k+qr(1-1)

這樣我們就能將 ⟨a,q⟩⋅⟨b,r⟩ 表示成 ⟨(abk+ar+bq)modp,qr⟩ 了！

而最後的結果 ⟨a′,q′⟩ 的答案就是 (a′+⌊kq′​⌋)modp。

不過，當 k>2 時，q′ 可能會變得超級大，我們勢必也需要在計算中對餘數做些處理。

而餘數可以再次表示成 ck+s 的形式貢獻到商裡面，所以 (1-1) 可以再變成如下算式：

(abk+ar+bq)k+qr=(abk+ar+bq+⌊kqr​⌋)k+(qrmodk)(1-2)

也就是說，

⟨a,q⟩⋅⟨b,r⟩=⟨(abk+ar+bq+⌊kqr​⌋)modp,qrmodk⟩

如此一來，我們也保證了餘數會在 [0,k) 的範圍內，兩數相乘也不會超出 64-bit 整數的範圍。

滿分解 2 --- 改變模數

首先，考慮如何計算 ⌊ka​⌋modp，即 ⌊ka​⌋ 除以 p 的餘數。

假設 ⌊ka​⌋ 除以 p 的商是 q、⌊ka​⌋ 除以 p 的餘數是 r，所求為 r。

⌊ka​⌋=qp+r(2-1)

根據除法定理，我們有

a=⌊ka​⌋⋅k+(amodk)(2-2)

將 (2-1) 代入 (2-2)：

a​=(qp+r)⋅k+(amodk)=qpk+rk+(amodk)​

為了得到 r，注意到

a≡rk+(amodk)(modpk)

由於 rk+(amodk)<rk+k=(r+1)k≤pk，這表示右式即為 a 除以 pk 的餘數：

amodpk=rk+(amodk)

注意到上式是 等號。由於 (amodk)<k，計算 r 的方法是直接將 rk+(amodk) 除以 k，商就是答案：

⌊krk+(amodk)​⌋=r+⌊kamodk​⌋=r

換句話說：

r=⌊kamodpk​⌋

回到原問題 ⌊kxy​⌋modp，根據上式可以得出：

⌊kxy​⌋modp=⌊kxymodpk​⌋

實作上，在計算 xy 的時候改成對 pk 取模，以避免 overflow。

但 pk 的最大值接近 long long 範圍的最大值，在快速冪的過程中需使用 __int128，才能避免兩數相乘時發生 overflow，得出錯誤的結果。

English Editorial

k=1 --- Binary Exponentiation

In class, we learned how to calculate xymodp in O(logy) time. For the subtask k=1 we can apply this method directly.

You can also find many related tutorial articles by searching "Binary Exponentiation" on Google.

The concept of binary exponentiation is to break down f(y):=xy into repeated subproblems solved recursively. Observe that f(y)=f(a)⋅f(y−a):

f(y)={f(⌊y/2⌋)⋅f(⌊y/2⌋)f(⌊y/2⌋)⋅f(⌊y/2⌋)⋅f(1)​if y is even,if y is odd.​

Obviously, there's no need to recursively compute f(⌊y/2⌋) twice, so it can be stored for direct use.

Binary Exponentiation - Recursive Version 
int fpow(int base, int exp, int mod) {
    if (exp == 0) return 1 % mod;
    int half = fpow(base, exp/2, mod);
    // return half * half % mod * (exp & 1 ? base : 1) % mod;
    if (exp % 2 == 0)
        return half * half % mod;
    else /// remember to take mod to reduce the maximum value during calculation ///
        return half * half % mod * base % mod;
    /*                     ^^^^^                                                  */
}

Here's another implementation that doesn't rely on recursion.

This implementation utilizes the binary representation of y, given by ∑i=0∞​2ibi​, where bi​∈{0,1}. This way, f(y) can be broken down into ∏i=0∞​f(2ibi​).

In C++, you can use y & (1 << i) to obtain bi​, and y >> i to get ⌊y/2i⌋. The example code continuously divides the exponent term by 2 using >>= 1 and extracts the "unit digit" value using & 1.

Binary Exponentiation - Binary Version 
int fpow(int base, int exp, int mod) {
    int ans = 1 % mod;
    while (exp) {
        if (exp & 1) ans = ans * base % mod;
        exp >>= 1, base = base * base % mod;
    }
    return ans;
}

Solution 1 --- Representing x as ak+q

To achieve a perfect score, an important observation needs to be made:

The multiple-of-k part doesn't matter when modulo p, but if the remainder part is taken modulo p, it will not contribute to the answer correctly.

Let's express the numbers to be multiplied as ⟨a,q⟩, representing ak+q, where 0≤q<k. Observe the result of multiplying two sets of numbers ⟨a,q⟩ and ⟨b,r⟩:

(ak+q)(bk+r)=abk2+(ar+bq)k+qr=(abk+ar+bq)k+qr(1-1)

This way, we can represent ⟨a,q⟩⋅⟨b,r⟩ as ⟨(abk+ar+bq)modp,qr⟩!

The final result ⟨a′,q′⟩ is then given by (a′+⌊kq′​⌋)modp.

However, when k>2, q′ might become very large, and we inevitably need to handle remainders in the calculations.

The remainder can once again be expressed as ck+s and contributes to the quotient, so (1-1) can be rewritten into the following equation:

(abk+ar+bq)k+qr=(abk+ar+bq+⌊kqr​⌋)k+(qrmodk)(1-2)

In other words,

⟨a,q⟩⋅⟨b,r⟩=⟨(abk+ar+bq+⌊kqr​⌋)modp,qrmodk⟩

This way, we also ensure that the remainder will be in the range [0,k), and the product of the two numbers will not exceed the range of a 64-bit integer.

Solution 2 --- Changing the Modulo

First, consider how to calculate ⌊ka​⌋modp, which is the remainder when ⌊ka​⌋ divided by p.

Assume that the quotient of ⌊ka​⌋ divided by p is q and the remainder of ⌊ka​⌋ divided by p is r, and what we are seeking is r.

⌊ka​⌋=qp+r(2-1)

According to the division theorem, we have

a=⌊ka​⌋⋅k+(amodk)(2-2)

Substitute (2-1) into (2-2):

a​=(qp+r)⋅k+(amodk)=qpk+rk+(amodk)​

To obtain r, note that

a≡rk+(amodk)(modpk)

Since rk+(amodk)<rk+k=(r+1)k≤pk, this means the right side is the remainder when a is divided by pk:

amodpk=rk+(amodk)

Note that the above equation is an equality. Since (amodk)<k, the way to calculate r is to directly divide rk+(amodk) by k, and the quotient is the answer:

⌊krk+(amodk)​⌋=r+⌊kamodk​⌋=r

In other words:

r=⌊kamodpk​⌋

Returning to the original problem ⌊kxy​⌋modp, according to the above equation:

⌊kxy​⌋modp=⌊kxymodpk​⌋

In practice, when calculating xy, it is modified to take modulo pk to avoid overflow.

However, the maximum value of pk is close to the maximum value of the long long range, and in the process of fast exponentiation, __int128 must be used to avoid overflow when multiplying two numbers, yielding incorrect results.

Solutions

Solution using x=ak+q 
#include <bits/stdc++.h>
using namespace std;

using i64 = int64_t;
using pii = pair<i64, i64>;

pii mul(pii a, pii b, i64 k, i64 p) {
    return {
        (a.first * b.first % p * k + a.first * b.second + a.second * b.first + a.second * b.second / k) % p,
        a.second * b.second % k
    };
}

pii fpow(pii base, i64 exp, i64 k, i64 p, pii ans = {0, 1}) {
    while (exp) {
        if (exp & 1) ans = mul(ans, base, k, p);
        exp >>= 1, base = mul(base, base, k, p);
    }
    return ans;
}

void solve() {
    i64 x, y, k, p; cin >> x >> y >> k >> p;
    
    cout << fpow({x/k, x%k}, y, k, p).first % p << "\n";
}

int32_t main() {
    ios_base::sync_with_stdio(0), cin.tie(0);
    
    int t; cin >> t;
    while (t--) solve();
    
    return 0;
}
Solution calculating ⌊kxymodkp​⌋modp 
#include <bits/stdc++.h>
using namespace std;

using i64 = int64_t;
using i128 = __int128;

i128 fpow(i128 base, i128 exp, i128 mod, i128 ans = 1) {
    while (exp) {
        if (exp & 1) ans = ans * base % mod;
        exp >>= 1, base = base * base % mod;
    }
    return ans;
}

void solve() {
    i64 x, y, k, p; cin >> x >> y >> k >> p;
    
    cout << i64(fpow(x, y, k*p) / k % p) << "\n";
}

int32_t main() {
