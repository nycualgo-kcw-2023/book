Subtask 2

在 n=1 的狀況下，u 會退化為一個純量，A 會退化成一個向量，問題退化成二個向量內積之最大值。

我們可以任意排列這二個向量內的元素，所以我們可以直接暴力列舉所有可能的排列，並記錄最大值。複雜度會是 O((m!)2)。

不過內積的計算只和哪個元素對上哪個元素有關，所以其實可以固定某一個向量，只需要列舉另外一邊的排列就可以了。

我們可以使用 C++ STL 函式 next_permutation 來幫助實作所有可能的排列。

using ll = long long;

時間複雜度為 O(m!)。

以下引理，將用以得出更有效率的解法：

Lemma

If x1​≥x2​ and a1​≥a2​, then x1​a1​+x2​a2​≥x1​a2​+x2​a1​

Proof:

To prove this, just expand (x1​−x2​)(a1​−a2​)≥0.

這個引理告訴我們，當我們將大數字與大數字相乘，小數字與小數字相乘時，我們將得到更大的結果。

我們可以得出一個結論：若我們將這二個向量排序，讓大的數字對上大的數字，此時答案會有最大值。

請注意 u 的正負號對答案的影響。

Subtask 3

下面是將要求的答案轉換為數學式的步驟：

uTAv​=(uTAv)11​=i=1∑n​(uT)1i​(Av)i1​=i=1∑n​ui​(j=1∑m​Aij​vj1​)=i=1∑n​j=1∑m​(ui​Aij​vj​)​

當 u 中的元素都是 1 時，式子變為 ∑i=1n​∑j=1m​Aij​vj​。

一樣可以使用大數乘以大數，小數乘以小數的想法，先將 v 從大到小（小到大）排序，再將 A 從大到小（小到大）排序後，從填滿第 1 個 column 開始，依序填滿直至第 m 個 column。

Sample Code 4 
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main()
{
    // Input...

    vector<ll> A_flat(n * m);
    for (int i = 0; i < n; ++i) {
        for (int j = 0; j < m; j++) {
            A_flat[i * m + j] = A[i][j];
        }
    }

    sort(v.begin(), v.end());
    sort(A_flat.begin(), A_flat.end());

    ll max_val = 0;
    for (int j = 0; j < m; j++) {
        for (int i = 0; i < n; ++i) {
            A[i][j] = A_flat[j * n + i];
            max_val += A[i][j] * v[j];
        }
    }

    // Output...
}

這段程式碼將 A 展平為一維向量，然後將 v 和 A 進行排序。接著，根據排序後的 v 將 A 填充為新的矩陣，最終計算並輸出最大值。

複雜度為 O(nmlog(nm))。

Subtask 4

由於可以重新排列所有元素，大多數人可能會考慮首先將向量 u 和 v 排序，然後再考慮如何排列矩陣 A 以獲得最佳結果。

然後，你可能會遇到一個情況，即矩陣 A 的左上角元素將與向量 u 和 v 中的較大數字相乘，而右下角元素將與二個較小的數字相乘。但如果向量 u 和 v 同時包含正數和負數，那麼左上角的元素將與正的大數字相乘，右下角的元素將負負得正…與正的大數字相乘？

因此，這個子任務提供了一個簡化的情況，其中所有數字都是非負整數，這樣你就不必擔心這個問題。

理論上，你可以將矩陣 A 的左上角填充為較大的數字，右下角填充為較小的數字，這應該會得到正確的答案。但是，右上角和左下角的元素應該如何填充？

實際上，我們並不知道如何填充這些元素，但我們希望你可以通過這個子問題來思考：究竟向量 u 和 v 中的數字將如何影響我們填充矩陣 A 的數字。

Solution

再看一次答案的式子：

uTAv​=i=1∑n​j=1∑m​ui​Aij​vj​=i=1∑n​j=1∑m​Aij​(ui​vj​)​

由於 Aij​ 可以任意排列，我們其實可以固定住 u 跟 v，然後根據所有的 ui​vj​ 的大小來重排 A 中的數字，讓 A 中大的數字跟大的 ui​vj​ 對上。

實作上可以先預處理 ui​vj​所有的乘積，然後對 A 中的數字進行排序。

複雜度為 O(nmlog(nm))。

Subtask 2

In the case where n=1, vector u degenerates into a scalar, and matrix A degenerates into a vector. The problem then simplifies to finding the maximum value of the dot product of two vectors.

Since the order of elements in these two vectors can be arbitrary, we can brute force all possible permutations and record the maximum value. The complexity is O((m!)2).

However, the calculation of the dot product depends only on which element corresponds to which element. So, we can fix one vector and enumerate the permutations of the other side.

We can use the C++ STL function next_permutation to help implement all possible permutations.

The following lemma will be used to derive a more efficient solution:

Lemma

If x1​≥x2​ and a1​≥a2​, then x1​a1​+x2​a2​≥x1​a2​+x2​a1​

Proof:

To prove this, just expand (x1​−x2​)(a1​−a2​)≥0.

This lemma tells us that when we multiply large numbers with large numbers and small numbers with small numbers, we get a larger result.

We can conclude that if we sort these two vectors, aligning larger numbers with larger numbers, the answer will have the maximum value.

Note the impact of the sign of vector u on the answer.

Subtask 3

Here are the steps to transform the required answer into a mathematical expression:

uTAv​=(uTAv)11​=i=1∑n​(uT)1i​(Av)i1​=i=1∑n​ui​(j=1∑m​Aij​vj1​)=i=1∑n​j=1∑m​(ui​Aij​vj​)​

When the elements of u are all 1, the expression becomes ∑i=1n​∑j=1m​Aij​vj​.

We can use the idea of multiplying large numbers by large numbers and small numbers by small numbers. First, sort vector v from large to small (or small to large), then sort matrix A from large to small (or small to large). Start filling from the first column and fill in order until all columns are filled.

Sample Code 4 
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main()
{
    // Input...

    vector<ll> A_flat(n * m);
    for (int i = 0; i < n; ++i) {
        for (int j = 0; j < m; j++) {
            A_flat[i * m + j] = A[i][j];
        }
    }

    sort(A_flat.begin(), A_flat.end());

    for (int j = 0; j < m; j++) {
        for (int i = 0; i < n; ++i) {
            A[i][j] = A_flat[j * n + i];

This code flattens matrix A into a one-dimensional vector, sorts vectors v and A, and fills matrix A based on the sorted order. Finally, it calculates and outputs the maximum value.

The complexity is O(nmlog(nm)).

Subtask 4

Since all elements can be rearranged, most people might consider sorting vectors u and v first and then think about how to arrange matrix A for optimal results.

You may encounter a situation where the upper-left element of matrix A will be multiplied by the larger numbers in vectors u and v, while the lower-right element will be multiplied by the two smaller numbers. But if vectors u and v contain both positive and negative numbers, the upper-left element will be multiplied by the positive large number, and the lower-right element will result in a positive product as well. It sounds weird, right?

Therefore, this subtask provides a simplified scenario where all numbers are non-negative integers, so you don't have to worry about this issue.

In theory, you can fill the upper-left of matrix A with larger numbers and the lower-right with smaller numbers, and this should yield the correct answer. However, how should the elements in the upper-right and lower-left be filled?

In reality, we don't know how to fill these elements, but we hope you can think about how the numbers in vectors u and v will affect filling the numbers in matrix A through this subproblem.

Solution

Let's revisit the formula for the answer:

uTAv​=i=1∑n​j=1∑m​ui​Aij​vj​=i=1∑n​j=1∑m​Aij​(ui​vj​)​

Since Aij​ can be arbitrarily rearranged, we can actually fix u and v and rearrange the numbers in A based on the sizes of all ui​vj​ to align the larger numbers in A with the larger ui​vj​.

In practice, you can preprocess all products of ui​vj​, then sort the numbers in A.

The complexity is O(nmlog(nm)).

By SorahISA 
#include <bits/stdc++.h>
using namespace std;

using int64 = long long;

int main() {
    ios_base::sync_with_stdio(0), cin.tie(0);
    
    int N, M; cin >> N >> M;
    
    vector<int64> U(N), V(M), A(N*M);
    for (int64 &x : U) cin >> x;
    for (int64 &x : V) cin >> x;
    for (int64 &x : A) cin >> x;
    sort(begin(A), end(A));
    
    /// ans = sum_i sum_j u[i] * A[i][j] * v[j] ///
    
    vector<int> id(N*M), id_inv(N*M);
    iota(begin(id), end(id), 0);
    sort(begin(id), end(id), [&](int x, int y) {
        int rx = x / M, cx = x % M;
        int ry = y / M, cy = y % M;
        return U[rx] * V[cx] < U[ry] * V[cy];
    });
    for (int i = 0; i < N*M; ++i) id_inv[id[i]] = i;
    
    stringstream sout;
    int64 ans = 0;
    for (int i = 0; i < N; ++i) sout << U[i] << " \n"[i == N-1];
    for (int j = 0; j < M; ++j) sout << V[j] << " \n"[j == M-1];
    for (int i = 0; i < N; ++i) {
        for (int j = 0; j < M; ++j) {
            sout << A[id_inv[M*i + j]] << " \n"[j == M-1];
            ans += U[i] * A[id_inv[M*i + j]] * V[j];
        }
    }
    
    cout << ans << "\n";
    cout << sout.str();
    
    return 0;
}
