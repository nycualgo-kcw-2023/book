Hint for Subtask 1: n≤9 (Released 2023.12.11)

Let fn​ denotes the number of ways to partition n people into several segments and gn​ denotes the number of accesses to cost matrix c, we have the transition formula as

fn​=2≤i≤n∑​fn−i​gn​=2≤i≤n∑​gn−i​+fn−i​⋅(2i​)​

and the base cases are fm​=[[m=0]] and gm​=0 for m≤1.

Therefore, if we fix the permutation, there are only f9​=21 ways to partition n people into several segments and g9​=277 accesses to the cost matrix.

Hint for Subtask 1: n≤9 (Released 2023.12.14)

To enumerate all the permutation, we can use the std::next_permutation function:

std::vector<int> perm(n);
// for (int i = 0; i < n; ++i) perm[i] = i;
std::iota(begin(perm), end(perm), 0); /// perm = [0, 1, 2, ..., n-1]
do {
    /* do something */
} while (std::next_permutation(begin(perm), end(perm)));

To enumerate all the ways to partition n people into several segments of size ≥2, we can use the following recursive function:

void recur(int n, int last, ...) {
    if (n == last) {
        /// all people are partitioned ///
    }
    else {
        for (int now = last + 2; now <= n; ++now) {
            /// [last, now) is in the same group ///
            recur(n, now, ...);
        }
    }
}

recur(n, 0, ...);
Hint for Subtasks 2, 3: n≤15 (Released 2023.12.18)

Let dp[S] where S⊆[n] denotes the minimum cost to assign all students in S to some group. We can loop through each T⊆[n] and update dp[S∪T] with dp[S]+cost(T).

A dumb way of implementation is to have n dimension of states: dpn dimensions[c1​][c2​][c3​][c4​][…][cn​]​​ where ci​∈{0,1} denotes the state that the ith student is assigned to some group.

  • But we must copy our code for each possible n, and the code would be extremely redundant.

We may also use an std::vector as the state, where we declare dp as std::map<std::vector, int>.

  • This implementation is better than the previous one, but it is still slow.

Observe that ci​∈{0,1}, does it remind you of anything?

Answer

Binary representation of integers.

We may use an integer in [0,2n) to represent the state, where the ith bit of the integer denotes the state of the ith student (whether i∈S).

Remark
  • We can use S & (1 << i) or (S >> i) & 1 to check whether the ith bit of S is on.
  • To check whether S is a subset of T, we can use (S & T) == S or (S | T) == T.
  • To get the complement set of S, we can use S ^ U, where U = (1 << n) - 1 is the mask of n 1's.
Hint for Subtasks 2, 3: n≤15 (Released 2023.12.29)

Let U={1,2,…,n}.
For S⊆U:
    For T⊆U:
        If ∣T∣≥2 and S∩T=∅:
            Update dp[S∪T] with dp[S]+cost(T).

In order for the above algorithm to work, we must ensure that each S is enumerated after we have used all S′⊊S to update dp[S].

The above algorithm works in O(4n⋅poly(n)) since the first and second for loops each has O(2n) iterations.

If the cost is precomputed, the algorithm can run in O(4n).

Hint for Subtasks 3, 4: n≤18 (Released 2023.12.29)

Suppose we can enumerate through only necessary T's, what is the time complexity of the below algorithm?

Let U={1,2,…,n}.
For S⊆U:
    For T⊆U∖S:
        If ∣T∣≥2:
            Update dp[S∪T] with dp[S]+cost(T).

Answer

Consider base-3 representation: the ith position is 0 if i∈S, 1 if i∈T, and 2 if i∈/S∪T. Each (S,T) pair can be represented by a unique base-3 number of length n.

Therefore, we can enumerate through all (S,T) pairs in O(3n).

Sample Code 
int U = (1 << n) - 1;
for (S be the subset of U) { /* and all subset of S is enumerated */
    for (int T = /* complement set of S */; T is non-empty; T = (T-1) & (/* complement set of S */)) {
        if (the number of 1s in T is >= 2) {
            dp[S union T] = min(dp[S union T], dp[S] + cost(T));
        }
    }
}

Line 3 in the above code is the key to enumerate through only necessary T. Please try to understand the code before you proceed.

You can check the bit one by one, or you may check out __builtin_popcount / std::popcount.
